Session 7: Deep Learning

Deep Learning vs Machine Learning

  • Deep Learning is a subset of machine learning.
  • Not exactly machine learning because it learns and uses data differently
  • Machine learning uses algorithms and pre-processing to organize data and define specific features, but deep learning is modeled to act similarly to the human brain so it automatically understands the data, understands which features to concentrate on, and there is no need for human intervention.

Deep Neural Network

  • A perceptron consists of one layer of TLU, but it is possible to have multiple layers connected to each other. With multiple layers, it can produce multiple outputs that classify the inputs. Essentially, it is a deep neural network when there are layers hidden deeply.
  • Forward and back propagation adjusts the weight and bias to better train the model by either moving forward and backwards through the layers

Deep Learning

  • Transfer Learning: Deep neural networks use a hierarchical architecture. If a model was already trained to do one task, it can apply what it learned to model another related task by using what it learned from the previous layers of the network and gathering new information with its layers higher up.
  • Deep learning is what we use for the latest technologies, especially with artificial intelligence applications and automated functionality
  • Deep neural networks and recurrent neural networks are deep-learning architectures
  • Typically deep learning explores deep layers but sometimes anything associated with neural networks will be considered deep learning
  • Although a neural network with just one hidden layer is powerful enough to model intricate functions, deep networks don’t need exponentially as many neurons so it can output better results more efficiently.
  • Extra hidden layers can help to make adjustments in improving the model

Determining Number of Neurons for Each Hidden Layer

  • To get the best results for the model, finding the optimal number of neurons and layers are crucial
  • While it’s pretty straightforward that the number of neurons depends on however needed for input and output, it’s still pretty challenging to find the “perfect number”
  • Can use the same number of neurons in all layers but can also change it up
  • Trial and error to add more and more neurons until it overfits
  • Typically, it’s easiest to just have a model with extra unnecessary layers and neurons
